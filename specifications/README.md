# Specifications and guidelines

Standards for builders so their products work well for uniocular users.

See the [roadmap](../ROADMAP.md) for planned specifications.

- **[Accessibility guidelines](accessibility-guidelines.md)** — how to
  build UIs and content that never require stereo vision.
- **[VR/FPV uniocular compatibility profile](vr-fpv-uniocular-compatibility.md)**
  — a device-class profile for VR/AR/MR headset and FPV-goggle manufacturers,
  with RFC 2119 requirements and A/AA/AAA conformance levels.
- Content guidelines — captioning depth, avoiding autostereoscopic-only
  media (planned).
- Hardware notes — cameras, mounts, and wearables (planned).

Specifications are written for implementers. Keep requirements testable
and use RFC 2119 keywords (MUST, SHOULD, MAY) where it adds clarity.
